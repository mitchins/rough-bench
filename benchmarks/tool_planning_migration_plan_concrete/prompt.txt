We need to migrate our user and account data from Firestore to PostgreSQL while keeping the API live.

Right now:
- Firestore holds user profiles, auth tokens, and account settings
- The API is stateless and reads/writes Firestore directly
- Traffic is steady but not huge: ~100 QPS average

The goal:
- Move data to PostgreSQL
- Update the API to read/write PostgreSQL
- Keep the service running during the migration
- Be able to roll back if something breaks

We also have concerns about data consistency during the transition. Some of the Firestore documents have nested fields that need to map to SQL tables. And we need to be careful about in-flight transactions.
