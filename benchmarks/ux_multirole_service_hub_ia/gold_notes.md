# Gold Notes

The core pressure here is role-context switching without identity fragmentation.

The key failure mode is dashboard soup: a response that sounds like a product
system but gives everything equal weight and never decides what matters right
now for a real person.

The strongest single diagnostic is the 7am scenario:

- the user has a client booking at 9am
- the same user has a worker shift at 10am
- the answer should surface one chronological morning, not two role-separated worlds

This task should reward:

- one account / one person with contextual surfaces
- task-based navigation
- operational surfaces distinct from management surfaces
- mobile justified by the reality of workers being mid-job or on the move
- desktop justified by planning and management work
- cognitive-load risks reduced by IA decisions, not visual-design claims

This task should punish:

- separate apps or separate accounts
- client mode / worker mode as the main mental model
- noun-heavy navigation with no "what do I do now" logic
- treating responsiveness as menu collapse
- answering in branding, UX-copy, or aesthetic language instead of structure
